الـ Anti-cheat والتصويت فكرة منطقية على الورق بس بالواقع:

المشكلة الحقيقية:

	•	أنت متحمس تلعب
	•	بعد كل لطشة لازم كل واحد يفتح تلفونه ويضغط ✅
	•	لو واحد ما ضغط → الكل يستنى
	•	هاذا بيكسر إيقاع اللعبة تماماً 😂

الحل الصح:
الـ Anti-cheat يكون اختياري وخفيف — مش إجباري بعد كل لطشة.

بدله اقترح:

	•	وضع عادي: بس واحد يسجل، بدون تصويت — سريع وسلس
	•	وضع مراجعة: في آخر الجلسة بس، كل اللاعبين يشوفوا ملخص الجولات ويقدروا يعترضوا
	•	زر “اعتراض” يظهر للكل طول الجلسة — لو حدا حس في غش يضغطه وقتها بس

هيك بتحل مشكلة الغش بدون ما تكسر إيقاع اللعبة.

المهمة ١ — تعديل نظام Anti-cheat جذري (أولوية قصوى)

المشكلة الحالية: التصويت بعد كل لطشة بيكسر إيقاع اللعبة — الناس متحمسة تلعب مش تصوت.

احذف التصويت الإجباري بعد كل لطشة نهائياً.

بدله اعمل هاذا النظام البسيط:

١. زر “🚨 اعتراض” ثابت ومرئي في شاشة الجلسة النشطة — صغير بالزاوية مش مزعج
٢. أي لاعب يحس في غش → يضغطه في أي وقت
٣. لما يضغط → يطلع modal بسيط:

"على شو بتعترض؟"
[آخر جولة] [جولة قبلها] [كتابة يدوية]


٤. بعد الاختيار → إشعار لباقي اللاعبين “⚠️ [اسم] عارض على [الجولة]”
٥. الكل يشوف ويقرروا مع بعض — التطبيق مش يتدخل
٦. في إعدادات الجلسة: toggle “تفعيل نظام الاعتراض” — افتراضي ON

احذف كلياً:

	•	مودال التصويت ✅/❌ بعد كل لطشة
	•	العداد التنازلي
	•	التأكيد التلقائي
	•	كل الكود المرتبط بالتصويت الإجباري

هاذا يحل مشكلة الغش بدون ما يكسر اللعبة. اكتب “✅ Anti-cheat اتعدّل” قبل ما تكمل.

المهمة ٢ — إضافة لعبة إستميشن

إستميشن لعبة شعبية جداً — 4 لاعبين فردي، 52 ورقة، 18 جولة.

واجهة تسجيل الجولة:

خطوة ١ — قبل الجولة (المزايدة):

كل لاعب يدخل طلبه (0-13):
[سهل:  ▼ 3 ▲]
[محمد: ▼ 4 ▲]
[خالد: ▼ 2 ▲]
[يوسف: ▼ 4 ▲]
المجموع: 13 ⚠️ (لازم ما يساوي 13)


	•	لو المجموع = 13 → أحمر + “المجموع ما يساوي 13، عدّل طلب واحد”
	•	داش (صفر لطشات): checkbox خاص “داش كول”

خطوة ٢ — بعد الجولة (النتائج):

كم لطشة أخذ كل لاعب:
[سهل:  ▼ 3 ▲] طلب: 3 ✅
[محمد: ▼ 5 ▲] طلب: 4 ❌
[خالد: ▼ 1 ▲] طلب: 2 ❌
[يوسف: ▼ 4 ▲] طلب: 4 ✅


حساب النقاط التلقائي:

function calcEstimationScore(bid, actual, isCall, isAlone, isDash) {
  const success = bid === actual
  
  if (success) {
    let score = 10 + actual
    if (isCall) score += 10      // صاحب الكول
    if (isAlone) score += 10     // الوحيد الفايز
    if (isDash && isOver) score += 25  // داش كول + أوفر
    if (isDash && isUnder) score += 33 // داش كول + أندر
    return score
  } else {
    let penalty = Math.abs(bid - actual)
    if (isCall) penalty += 10
    if (isAlone) penalty += 10
    if (isDash && isOver) penalty += 25
    if (isDash && isUnder) penalty += 33
    return -penalty
  }
}


ملاحظات الواجهة:

	•	اعرض “جولة X من 18” في الهيدر
	•	اعرض نوع الجولة: “جولة عادية” (1-13) أو “كولور” (14-18)
	•	اللعبة تنتهي تلقائياً بعد 18 جولة
	•	الفائز = أعلى نقاط

المهمة ٣ — إضافة لعبة كسرة

كسرة = مزيج تركس كومبلكس + طرنيب — 4 لاعبين فردي، 52 ورقة، 4 ممالك.

كل مملكة = 4 ألعاب بالترتيب اللي يختاره صاحبها:

اللعبة ١ — كومبلكس:

	•	نفس قواعد كومبلكس: يخسر على ديناري (-10)، بنات (-25)، شيخ الكبة (-75)، لطش (-15)
	•	التدبيل متاح لشيخ الكبة والبنات

اللعبة ٢ — طرنيب:

	•	صاحب المملكة يختار الطرنيب
	•	+10 نقاط لكل لطشة يأخذها

اللعبة ٣ — لطشين:

	•	الكل يتجنب آخر لطشتين:
	•	اللطشة 12: -60 نقطة
	•	اللطشة 13: -70 نقطة

اللعبة ٤ — تركس:

	•	نفس نظام تركس: +200/+150/+100/+50

واجهة التسجيل:

المملكة الحالية: سهل
اللعبة: [كومبلكس ✓] [طرنيب] [لطشين] [تركس]
← صاحب المملكة يرتبهم بالسحب


بعد كل لعبة من الـ 4 → التطبيق ينتقل للتالية تلقائياً.
بعد الـ 4 → المملكة تنتقل لليمين.

المهمة ٤ — إضافة لعبة كوت بو

كوت بو لعبة شعبية جداً بالخليج والأردن — فريقين، بدون طرنيب.

الأساسيات:

	•	4 لاعبين، فريقين، 52 ورقة
	•	الهدف: فريق يأخذ 7 لطشات أو أكثر من 13 = “كوت”

واجهة التسجيل:

من أخذ الكوت؟
[فريق A — X لطشة] [فريق B — Y لطشة]

النتيجة:
فريق A أخذ 8 → كوت ✅ +1 نقطة
فريق B أخذ 5 → خسر

أو:
فريق A أخذ 13 → كوت بو كامل ✅ +2 نقطة


حساب النقاط:

	•	كوت عادي (7-12 لطشة): +1 نقطة للفريق الفايز
	•	كوت بو (13 لطشة): +2 نقطة
	•	الفريق اللي يخسر: 0

الفوز: أول فريق يوصل 7 نقاط يفوز (قابل للتخصيص: 7/11/14)
