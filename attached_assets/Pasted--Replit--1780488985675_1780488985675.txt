المهمة ١ — حفظ الجلسات الأونلاين بـ Supabase (أولوية قصوى)

المشكلة: الجلسات الأونلاين محفوظة بالذاكرة فقط — لو السيرفر أُعيد تشغيله كل الجلسات تضيع.

المطلوب:

أولاً شغّل هاذا الـ SQL بـ Supabase SQL Editor:

create table if not exists public.online_sessions (
  id text primary key,
  code text unique not null,
  game_id text not null,
  game_variant text default 'normal',
  host_id uuid references auth.users(id) on delete cascade,
  status text default 'waiting',
  settings jsonb default '{}',
  players jsonb default '[]',
  rounds jsonb default '[]',
  target_score integer,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);

alter table public.online_sessions enable row level security;
create policy "online_sessions_all" on public.online_sessions for all using (true);

create or replace function update_updated_at()
returns trigger as $$
begin
  new.updated_at = now();
  return new;
end;
$$ language plpgsql;

create trigger online_sessions_updated_at
  before update on public.online_sessions
  for each row execute function update_updated_at();


ثانياً عدّل كل العمليات اللي كانت تشتغل على الـ Map في الذاكرة عشان تشتغل على Supabase:

	•	إنشاء جلسة → INSERT بـ Supabase
	•	انضمام لاعب → UPDATE الـ players array
	•	تسجيل جولة → UPDATE الـ rounds array
	•	جلب الجلسة → SELECT by code
	•	إنهاء الجلسة → UPDATE status = ‘finished’

ثالثاً تأكد إن الـ polling كل 2 ثانية يجيب البيانات من Supabase مباشرة مش من الذاكرة.

اكتب “✅ المهمة ١ خلصت — الجلسات محفوظة بـ Supabase” قبل ما تكمل.

المهمة ٢ — تخلي كل لاعب يسجل النقاط (مش المضيف بس)

المشكلة: هلق بس المضيف يقدر يسجل — باقي اللاعبين مشاهدون فقط.

المطلوب:

١. كل لاعب منضم للجلسة يشوف نفس شاشة التسجيل مش شاشة مشاهدة
٢. أي لاعب يقدر يضغط “سجّل جولة” — مش المضيف بس
٣. لما أي لاعب يسجل جولة → تتحدث عند الكل فوراً
٤. أضيف حقل recorded_by لكل جولة عشان نعرف مين سجّل
٥. الـ Anti-cheat (لو مفعّل) يطلب تأكيد من باقي اللاعبين بعد أي تسجيل

منطق التحكم:

// كل لاعب يقدر يسجل — مش بس المضيف
const canRecord = session.players.some(p => p.id === currentUser.id)


اكتب “✅ المهمة ٢ خلصت — كل لاعب يقدر يسجل” قبل ما تكمل.

المهمة ٣ — Anti-cheat على الأونلاين

المشكلة: Anti-cheat شغال محلي بس — على الأونلاين ما يشتغل.

المطلوب:

١. لما لاعب يسجل جولة وAnti-cheat مفعّل → احفظ الجولة بـ status = ‘pending’
٢. باقي اللاعبين يشوفوا مودال التصويت تلقائياً (عبر الـ polling أو Realtime)
٣. كل لاعب يصوت ✅ أو ❌ من تلفونه
٤. المنطق:

// عداد الأصوات
const majority = Math.floor(players.length / 2) + 1

if (approvals >= majority) {
  // سجّل الجولة — status = 'confirmed'
  // flash أخضر عند الكل
} else if (rejections >= majority) {
  // ألغِ الجولة — status = 'rejected'  
  // flash أحمر + رسالة عشوائية
}


٥. Timeout: لو ما وصل أغلبية بالوقت المحدد → تأكيد تلقائي
٦. الرسائل عند الاعتراض (عشوائية):

	•	“في خلاف! ناقشوا مع بعض 😂”
	•	“مين اللي بيغش هون؟ 👀”
	•	“بتشك فيك والله 😤”
	•	“يا زلمة اتفقوا!”

اكتب “✅ المهمة ٣ خلصت — Anti-cheat شغال أونلاين” قبل ما تكمل.

المهمة ٤ — واجهات نقاط مخصصة للألعاب الـ 9 الناقصة

هاذي الألعاب كلها هلق بتستخدم Generic Entry (إدخال يدوي بدون حساب تلقائي). المطلوب تعمل واجهة مخصصة لكل وحدة بالحساب التلقائي الصح.

٤أ — بلوت (Baloot)

	•	فريقين، الهدف 152
	•	كل جولة: اختار الفريق الفائز بالجولة + النقاط
	•	آخر لطشة: +10 للفريق اللي أخذها (checkbox)
	•	إعلان السلسلة (سلسلة): أضيف نقاط إضافية للفريق المعلن
	•	لو فريق وصل 152 أو أكثر → فاز

٤ب — بصرة (Basra)

	•	لاعبين أو فريقين، الهدف 121
	•	بعد كل توزيعة اختار:
	•	مين أخذ أكثر أوراق (27+) → +30
	•	مين أخذ آس → +1 لكل آس
	•	مين أخذ جاك → +1 لكل جاك
	•	مين أخذ 2 كلوب → +2
	•	مين أخذ 10 ديناري → +3
	•	كم بصرة عمل كل لاعب → +10 لكل بصرة
	•	زر سريع “بصرة! 🎉” يضيف +10 فوراً

٤ج — ليخة (Leekha)

	•	بعد كل جولة لكل لاعب:

	•	checkbox “Shoot the Moon” (أخذ كل القلوب + بنت سباتي) → هو يأخذ 0، الباقين +26
	•	لو وصل لاعب 101+ → خرج من اللعبة، أقل نقاط يفوز
